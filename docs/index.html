Pull cards into a workspace, annotate the relationship, then write a real double-link back only when the connection earns its place in the vault.

A quieter tool for serious linked thinking.

Plain Markdown vaults

Your source of truth is ordinary Markdown. SQLite is only a fast local index.

Box-based structure

Top-level numbered cards act as boxes. Their Folgezettel descendants form local contexts.

Real double-links

Explicit [[link]] relations are shown clearly without recursively flooding the canvas.

Potential links

Unlinked references and high-confidence local similarity appear as optional suggestions.

Workspace canvases

Use temporary cards, notes, relation labels, and apply-to-vault links when ready.

Graph focus

Explore global structure through Box, Link, and Tag relations with first-degree highlighting.

Saved layouts

Manual placement is preserved per box and workspace, with reset layout when you need it.

Canvas image export

Export the visible box or workspace as a PNG with clean solid links and safe margins.

Local plugin path

Trusted local plugins can extend commands, UI, and export workflows through the early SDK.

1.5

Folium 1.5 is the first version shaped around the final interaction model: lighter main canvas, clearer workspace semantics, stronger visual identity, and fewer confusing filters.

From feature-heavy 1.0 to usable 1.5.

Folium 1.0 proved the core platform: local vaults, Electron packaging, workspaces, graph view, attachments, backups, and plugin groundwork. It was powerful, but some interactions were too eager.

  • 1.5 keeps box structure visible while stopping imported link cards from expanding endlessly.
  • Potential links are now an optional assistant layer, not a second structure system.
  • GraphView was reduced to the concepts users actually think with: Box, Link, and Tag.
  • Workspaces now better separate temporary reasoning from durable vault links.
