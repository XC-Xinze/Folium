# 严格结构性 box

在 sidebar 点 [[1]] → 画布显示 1 整棵子树（[[1a]] [[1a1]] [[1a2]] [[1b]] [[1b1]] [[1c]]）。

点 [[1a]] → 画布**只**显示 1a 子树（[[1a]] [[1a1]] [[1a2]]）。不会包含 [[1b]] 因为它不在 1a 子树里。

## 为什么不靠 INDEX 卡的 [[crossLinks]]

旧版本设计：INDEX 用 [[crossLinks]] 手动 curate "我这个 box 装哪些卡"。问题：
- 用户得记得维护
- 一张卡可能"该归"多个 box，列哪个？
- 跟 Folgezettel 命名重复，违反 DRY

现在 box 完全派生自 Folgezettel id。一致、自动、零维护。

详见 [[1c]] 链接策略 —— cross-link 现在只是普通引用，不当 box 成员标记用。
