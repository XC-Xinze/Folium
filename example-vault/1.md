# Zettelkasten 方法论

这是一个**顶级盒子**（top-level box）。因为它有子卡（[[1a]]、[[1b]]、[[1c]]），系统派生出 INDEX 状态 —— 不需要你手动标。

## 核心原则

- **Atomic notes**：一张卡一个想法，详见 [[1b]]
- **Folgezettel**：用 id 表达父子关系，详见 [[1a]]
- **Linking is the killer feature**：而不是 folder 层级
- **Tags are orthogonal**：跟 Folgezettel 树正交，跨结构联系想法

## 关于 INDEX

INDEX 不是你"选择"的，而是**结构涌现**：只要这张卡下面有子卡（按 Folgezettel 命名），它自动是 INDEX。这才是卢曼原版的精神：先写卡，结构自然生长。

#zettelkasten 是一种**思考工具**，不是分类系统。
