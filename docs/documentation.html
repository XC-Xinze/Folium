Method guide

Using Luhmann-style cards in Folium.

Folium treats notes as small, linkable cards. The goal is not to collect everything, but to create a system where each card can point to the next useful thought.

Core principles

A Luhmann-style card system is built from atomic notes, stable identifiers, and explicit links. A card should usually hold one idea, written in your own words, with enough context to make sense when it appears outside the source you read.

  • One card, one useful thought. Keep it small enough to link precisely.
  • Write in your own words. A card is not a highlight. It is a processed idea.
  • Link for a reason. Explain why two cards belong near each other.
  • Prefer growth over categories. Structure should emerge from use.
Folium keeps the source as Markdown. The database is an index, not the owner of your notes.

Daily workflow

  1. Capture an idea as a short card. Do not over-organize it at the moment of capture.
  2. Give the card a stable ID and a plain title.
  3. Add links to cards that help explain, challenge, extend, or contextualize the idea.
  4. Use chain view to read along the thought path.
  5. Use workspaces when the relationships are still provisional.

Numbering

Folium uses Luhmann-style IDs such as 1, 1a, 1a1, and 1a1b. The number is not a folder path. It is a position in a growing sequence.

  • 1 can be a broad index card.
  • 1a continues or branches from 1.
  • 1a1 develops 1a further.
  • Top-level numbered cards are treated as index boxes in Folium.

Workspace use

A workspace is the place for thinking before commitment. Bring cards together, draw relations, add notes, and create temporary cards. When a relation becomes durable, apply it back to the vault as a real double-link.

This separation keeps the vault clean while still allowing messy visual reasoning.

核心原则

鲁曼式卡片系统由原子化卡片、稳定编号和明确链接组成。一张卡通常只承载一个想法，并且要用自己的语言写出来， 让它脱离原始材料之后仍然能够被理解。

  • 一张卡，一个有用想法。 卡片越小，链接越精确。
  • 用自己的话写。 卡片不是摘抄，而是被你处理过的思想。
  • 带着理由链接。 不只是连起来，还要知道为什么它们应该靠近。
  • 让结构自然生长。 不要一开始就用分类把思想锁死。
Folium 的源文件是 Markdown。SQLite 只是索引，不是你的笔记本体。

日常流程

  1. 先把想法写成一张短卡，不要在捕捉阶段过度整理。
  2. 给卡片一个稳定编号和清晰标题。
  3. 把它链接到能够解释、反驳、延伸或补充语境的卡片。
  4. 用链式视图沿着思路阅读。
  5. 关系还不确定时，先放进 workspace 里实验。

编号方式

Folium 使用类似 1、1a、1a1、1a1b 的编号。 编号不是文件夹路径，而是思想序列里继续生长的位置。

  • 1 可以是一张大的索引卡。
  • 1a 从 1 继续或分叉。
  • 1a1 进一步展开 1a。
  • Folium 会把顶层数字卡默认看成 index box。

Workspace 用法

Workspace 是思想还没有定型之前的地方。你可以把卡片拖进来，画关系，加备注，创建临时卡。 当某条关系真的值得保留时，再 apply 回 vault，变成真正的双链。

这样可以让主仓库保持干净，同时保留视觉化、临时化、实验性的思考过程。
