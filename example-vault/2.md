# ZettelkastenCard

跟 Obsidian / Logseq 的差异化：**我们更卢曼原教旨**。

## 数据模型

- `.md` 文件是**真源**（source of truth），跟 Obsidian 一样可用任何编辑器开
- SQLite 是**派生索引**（derived），可随时重建（Settings → Rebuild Index）
- 一个机器可有多个 vault，sidebar 底部 picker 切换

## 核心设计选择（详见子卡）

- [[2a]] INDEX 自然生长 —— 不是用户选择，结构涌现
- [[2b]] Box = Folgezettel 子树 —— 严格按 id 结构，不依赖 INDEX 手动 curate
- [[2c]] Workspace = 临时画布 —— 既不污染 vault 又能孵化新分支

## 操作惯例

- [[2d]] 拖动语义全统一 —— id 徽章 = 拖出去链接 / reparent / 进 workspace
- [[2e]] Trail tag + 单层 potential —— Canvas 的心智模型
